• ChannelContext

    一个连接对应一个ChannelContext对象，ChannelContext是一个抽象类，他的两个子类是ServerChannelContext和ClientChannelContext，如果你是用tio作服务器，那么就ServerChannelContext，反之则是ClientChannelContext。

    该类包含了诸如编码任务对象，解码任务对象，执行业务的处理任务对象，还有连接的监控统计数据ChannelStat

  • GroupContext

    一个监听端口对应一个GroupContext对象，GroupContext是一个抽象类，他的两个子类是ServerGroupContext和ClientGroupContext，如果你是用tio作服务器，那么就ServerGroupContext，反之则是ClientGroupContext。

    该类包含了诸如AioHandler对象，AioListener对象，还有监听端口的监控统计数据ServerGroupStat

  • ObjWithLock

    顾名思义，这是个自带锁的对象，锁自然是锁依附于这个对象的对象。网络编程，并发同步问题随处可见，没一把好的锁真是寸步难行，所以t-io创造了这么一个对象出来，阅读过t-io源代码的同学，一定对这个对象印象深刻，它几乎是无处不在！

    为了更简便地使用，该对象有几个子类可以使用，分别是：ListWithLock，MapWithLock，SetWithLock
